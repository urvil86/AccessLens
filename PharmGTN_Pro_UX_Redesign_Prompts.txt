How to Use This DocumentThis document contains 10 sequential prompts designed to be fed one at a time to Claude Code in VS Code. Each prompt builds on the output of the previous one. The prompts will transform the existing PharmGTN Pro app from a 7-tab expert tool into a streamlined 4-page application optimized for market access teams working on biosimilar forecasting.Run them in order (Prompt 0 through Prompt 9). After each prompt, review the output, run the dev server with npm run dev to verify, and fix any issues before moving to the next. Prompt 0 is restructuring, Prompts 1 through 6 build individual features, Prompt 7 ties navigation together, Prompt 8 adds Excel integration, and Prompt 9 is the polish pass.Current Architecture (What You Have)Stack: React 19, TypeScript, Vite 8, Tailwind CSS v4, Zustand 5, Recharts 3, Lucide ReactFiles: 7 tab components (DashboardTab, ForecastTab, ChannelTab, ContractTab, ASPTab, GTNTab, BuyBillTab), 7 shared components, Zustand store (useStore.ts), compute engine (compute.ts), types, constants, hooksKey UX Issues: 7 tabs require sequential navigation. Individual NumberInput steppers for every cell (year x metric) are tedious. No Excel upload for forecast volumes. No scenario management or version control. No dedicated sensitivity analysis. ASPTab is 592 lines with stress + rescue simulators crammed in. No data export. No guided onboarding.Target Architecture (What You Will Get)4 pages instead of 7 tabs: Dashboard, Assumptions (unified), Results (ASP + GTN combined), Scenarios and SensitivitySpreadsheet-style editing using editable table grids instead of hundreds of individual NumberInput componentsExcel integration for uploading volume forecasts and downloading assumption snapshotsScenario manager with save/compare/lock/version-control for assumption setsSensitivity tools including tornado diagrams, two-way heat maps, and the existing stress/rescue simulators

label: string; }[];               // e.g. [{key: "com_pbm", label: "Commercial PBM"}, ...]    data: Record<string, Record<string, number>>;           // data[rowKey][colKey] = value    onChange: (rowKey: string, colKey: string, value: number) => void;    validation?: {                                          // optional per-column validation      type: 'sum-to-100' | 'range';      min?: number;      max?: number;    };    suffix?: string;                                        // e.g. "%" shown after each value    readOnly?: boolean;    formatValue?: (v: number) => string;                    // for display mode    step?: number;                                          // increment for arrow keys, default 0.5    decimalPlaces?: number;                                 // default 1  }Behavior:- Renders as a table with sticky first column (row labels) and sticky header row (column labels)- Cells are editable: click to type, Tab/Enter to move to next cell, arrow keys navigate- When validation.type is 'sum-to-100': show a footer row with column sums, green if 99.5-100.5%, red otherwise- When validation.type is 'range': highlight cells outside min/max in amber- Support paste from Excel: user selects a cell, pastes, and data fills rightward and downward from that cell- Support "Fill Right" context action: right-click a cell, choose "Copy to all years", fills that value across the row- Keyboard: Escape cancels edit, Enter commits, Tab moves right, Shift+Tab moves leftStyling (using existing Tailwind theme):- Header: bg-[#004567] text-white text-xs font-mono- Cells: bg-white border border-gray-200, on focus: ring-2 ring-[#C98B27]- Sticky first column: bg-[#F7F9FC] font-semibold text-[#004567]- Validation error cells: bg-amber-50 border-amber-300- Footer sum row: bg-gray-50 font-mono font-boldAlso update src/shared/index.ts to export this new component alongside the existing ones.Do NOT integrate this into any page yet Ń just build and export the component.
